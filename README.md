# Assignment 1 - Grobid analysis of open-access papers

## Overview
This project analyzes 10 open-access research articles using Grobid.

The project generates:
- a keyword cloud based on the abstracts
- a visualization of the number of figures per article
- a list of links found in each paper

## Dataset
The dataset consists of 10 open-access PDF papers related to open science, reproducibility and research software engineering.

## Repository structure
- `data/pdfs/`: input PDF files
- `data/tei/`: TEI/XML files extracted with Grobid
- `src/`: source code
- `results/`: generated outputs
- `tests/`: tests

## Outputs
The project generates the following files:
- `results/abstracts.csv`
- `results/figures_per_paper.csv`
- `results/figures_per_paper.png`
- `results/links_per_paper.csv`
- `results/keyword_cloud.png`

## How to run
1. Place the input PDF files in `data/pdfs/`
2. Extract TEI/XML files with Grobid into `data/tei/`
3. Run:
   - Run the parser:`py src/make_parse_tei.py`
   - Generate the figure-count plot:`py src/figures_plot.py`
   - Generate the keyword cloud:`py src/make_wordcloud.py`

   ## Validation

### Abstract extraction and keyword cloud
The abstracts were extracted from the TEI/XML files generated by Grobid and stored in `results/abstracts.csv`. To validate this step, a sample of extracted abstracts was manually compared against the abstracts visible in the original PDF files. The keyword cloud was then generated only from the non-empty abstracts in that CSV file. This validation confirms that the word cloud reflects the abstract text actually present in the selected papers.

### Figure count per article
The number of figures per paper was computed from the `<figure>` elements found in each TEI/XML file and stored in `results/figures_per_paper.csv`, which was then used to generate the corresponding plot. To validate this result, several papers were manually checked by comparing the automatic figure count with the figures visible in the original PDF files. Minor differences may appear when the PDF structure affects Grobid parsing or when figure-like elements are inconsistently detected.

### Link extraction
Links were extracted from the TEI/XML files using both XML attributes and URL patterns detected in the text. The final results were stored in `results/links_per_paper.csv`. To validate this step, links from a sample of papers were manually checked against the original PDF files to confirm that the extracted URLs were actually present in the documents. Duplicate links were removed while preserving the original order of appearance.

In all three cases, validation was performed through manual comparison with the original PDF documents, since they are the ground truth source for the information processed by Grobid.
## Limitations
Grobid extraction quality depends on the structure and quality of each PDF.
Some papers may have incomplete metadata extraction, such as missing titles or abstracts.
Figure counts depend on the figure elements detected in the TEI/XML output.
Link extraction may miss some links or include links found in references and notes.
The dataset is limited to 10 papers, so the results are only illustrative.
