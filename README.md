# Artificial-Intelligence-And-Open-Science-In-Research-Software-Engineering

# Assignment 1 - Grobid analysis of open-access papers

## Overview
This project analyzes 10 open-access research articles using Grobid.

The project generates:
- a keyword cloud based on the abstracts
- a visualization of the number of figures per article
- a list of links found in each paper

## Dataset
The dataset consists of 10 open-access PDF papers related to open science, reproducibility and research software engineering.

## Repository structure
- `data/pdfs/`: input PDF files
- `data/tei/`: TEI/XML files extracted with Grobid
- `src/`: source code
- `results/`: generated outputs
- `tests/`: tests

## Outputs
The project generates the following files:
- `results/abstracts.csv`
- `results/figures_per_paper.csv`
- `results/figures_per_paper.png`
- `results/links_per_paper.csv`
- `results/keyword_cloud.png`

## Current status
The PDF to TEI/XML extraction pipeline has been completed using Grobid.
The XML files were parsed to extract abstracts, figure counts and links.
A figure-count visualization and a keyword cloud have already been generated.

## Installation
Install the required Python packages with:
   py -m pip install -r requirements.txt

## How to run
1. Place the input PDF files in `data/pdfs/`
2. Extract TEI/XML files with Grobid into `data/tei/`
3. Run:
   - Run the parser:`py src/parse_tei.py`
   - Generate the figure-count plot:`py src/make_figures_plot.py`
   - Generate the keyword cloud:`py src/make_wordcloud.py`

## Validation
## Abstract extraction and keyword cloud
The abstracts were extracted from the TEI/XML files generated by Grobid.
To validate this step, several extracted abstracts were manually compared with the abstracts visible in the original PDF files.
The keyword cloud was generated only from the non-empty abstracts stored in results/abstracts.csv.

## Figure count per article
The number of figures was computed from the figure elements found in each TEI/XML file.
To validate this result, several papers were manually checked by comparing the generated figure count with the figures visible in the original PDF.
Some differences may appear when PDF structure affects Grobid parsing.

## Link extraction
Links were extracted from the TEI/XML files using both XML attributes and URL patterns found in the text.
To validate this step, extracted links from several papers were manually checked against the original PDF files.
Duplicate links were removed while preserving the original order of appearance.

## Limitations
Grobid extraction quality depends on the structure and quality of each PDF.
Some papers may have incomplete metadata extraction, such as missing titles or abstracts.
Figure counts depend on the figure elements detected in the TEI/XML output.
Link extraction may miss some links or include links found in references and notes.
The dataset is limited to 10 papers, so the results are only illustrative.
